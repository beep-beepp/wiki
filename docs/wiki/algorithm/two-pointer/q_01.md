# 문제

N개의 수로 이루어진 수열이 주어집니다. </br>
이 수열에서 연속부분수열의 합이 특정숫자 M이 되는 경우가 몇 번 있는지 구하는 프로그램을 작성하세요.</br>

만약 N=8, M=6이고 수열이 다음과 같다면

1 2 1 3 1 1 1 2

합이 6이 되는 연속부분수열은 {2, 1, 3}, {1, 3, 1, 1}, {3, 1, 1, 1}로 총 3가지입니다.


입력 </br>
첫째 줄에 N(1≤N≤100,000), M(1≤M≤100,000,000)이 주어진다.
수열의 원소값은 1,000을 넘지 않는 자연수이다.


출력 </br>
첫째 줄에 경우의 수를 출력한다.

예시 입력 </br>
8 6 </br>
1 2 1 3 1 1 1 2

예시 출력 </br>
3

## 후기
처음에 생각 없이 무지성으로 이중 for문을 사용해서 그런지 시간복잡도가 $O(N^2)$이 되어 시간초과로 계속 틀리게 됨. 해당 문제는 **투 포인터(Two Pointers)** 알고리즘을 사용하면 효율적으로 해결 가능

### 투 포인터 (Two Pointers)
- 배열의 원소들이 모두 **자연수(양수)**이기 때문에 가능한 전략.
- 구간을 늘리면 합이 커지고, 구간을 줄이면 합이 작아지는 성질을 이용.
- lt (왼쪽 포인터)와 rt (오른쪽 포인터) 두 개의 변수를 사용하여 탐색 범위를 조절.


### 알고리즘 로직
- lt와 rt를 모두 0에서 시작합니다. sum은 0으로 초기화.
- rt를 오른쪽으로 이동시키며 sum에 값을 더하기. (구간 확장)
- 만약 sum이 M보다 크다면? -> sum이 M 이하가 될 때까지 lt를 오른쪽으로 이동시키며 sum에서 값 빼기. (구간 축소)
- 만약 sum이 M과 같다면? ->count를 1 증가.
- rt가 배열 끝까지 갈 때까지 반복.



<details>
  <summary>📌 정답</summary>
  
  ``` java
    public int solution (int n, int[] arr){
            int answer = 0;
            int sum = 0;
            int lt = 0;

            for(int rt = 0; rt < n/2+1; rt++){
                sum += arr[rt];

                if(sum == n){
                    answer++;
                }

                while(sum >= n){
                    sum -= arr[lt++];
                    if(sum == n){
                        answer++;
                    }
                }

            }

            return answer;
        }
```

## 코드 동작 원리 (예시: 1 2 1 3 1 1 1 2, M=6)
- rt가 이동하며 더하다가 1, 2, 1, 3이 되면 합이 7이 됨. (M보다 큼)
- while문 동작: lt가 가리키는 맨 앞의 1 빼기. 합은 6이 됨.
- 합이 6이므로 count를 증가. ({2, 1, 3} 발견)
- 다음 rt가 이동하여 1을 더하면 합은 7이 됨.
- 다시 lt를 이동시켜 2를 빼기. 합은 5가 됨. (M보다 작으므로 멈춤)
- 이 과정을 끝까지 반복.
